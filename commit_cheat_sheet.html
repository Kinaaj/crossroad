Rules & Structure

Atomic Commits One logical change per commit. The "AND" Rule If you write "and" in the message, split the commit. Imperative Mood Use "Add feature" not "Added feature". Structure type: Subject
Body (optional)

Commit Types

feat New feature (e.g., login form) fix Bug fix (e.g., fix crash) docs Documentation only (README) style Formatting, semi-colons (no code change) refactor Rewrite code without changing behavior test Adding or fixing tests/test data

Examples

Database feat: Define initial DB schema Frontend style: Fix alignment in nav bar Logic fix: Validate user email input Cleanup chore: Remove unused images ‚ùå BAD fixed bugs, wip, update css

Workflow

1. Status git status - See what changed 2. Stage git add filename - Pick files 3. Commit git commit -m "feat: ..." 4. Push git push - Send to server 5. Undo (Soft) git reset --soft HEAD~1
